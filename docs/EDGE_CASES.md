# napflow — Edge Cases: Resolution Ledger

EC01–EC41 were previously marked resolved. The 2026-07-11 first-working-
version review **reopened EC09, EC10, EC22, EC27, EC32, EC35, and EC38**
and added EC42–EC51. M1 closed EC38, EC46, and EC51 on 2026-07-12; M2
closed EC09, EC43, and EC48 on 2026-07-13; M4 closed EC32 and EC50, M5
closed EC47, and M6 closed EC42 and EC49 the same day. M7 closed EC44 and
added then fixed EC53–EC54 on 2026-07-14. Four cases remain open: EC10,
EC22, EC27, and EC35; EC27's cooperative half is complete.
Documentation or explicit risk acceptance alone does not mean the
underlying behavior is resolved. The full original analyses for
EC01–EC23 are preserved in `archive/EDGE_CASES.md`.

Resolution kinds: **ADR** (a `DECISIONS.md` entry), **fix** (behavior
and regression test landed), **doc** (behavior acceptable but needed an
explicit statement), **PARTIAL** (one independently tested half landed),
**OPEN** (planned owner/milestone; not resolved).

| EC01 | Unreached declared End port → run reported `passed` (false green) | ADR **D18**: End ports `required` by default; unwritten required port at quiescence ⇒ `failed`, exit 1. Schema *Start/End rules*, engine §2. |
| EC02 | Unconnected `exhausted`/`expired` semantics self-contradictory | ADR **D19**: guard exhaustion ports are ordinary pass-through outputs, not error ports; unconnected = dropped + lint **W106**; "gave up" fails via D18 or a wired assert. Schema *Wire format* + *Guards*; example wires `attempts.exhausted`. |
| EC03 | `merge: all` in a cycle with one re-firing input → stall     | doc: `all` is a one-shot rendezvous (slots clear on emit); use `any` to rejoin retry paths. Schema *Merge*, engine §4. |
| EC04 | Multi-input node with a never-arriving input → silent skip   | doc: "skipped" is a first-class outcome; escalate to failure via required End ports (D18). Engine §4. |
| EC05 | Cross-frame aggregation of asserts/errors unstated           | ADR **D20**: outcomes aggregate run-wide (worst state anywhere in the frame tree); only data is frame-isolated. Engine §2. |
| EC06 | Loop `on_error`/`errors`: "iteration error" undefined        | ADR **D20**: iteration error = body frame ending `failed`/`error`; `on_error` only gates further scheduling; failures always score. Engine §5, schema *Loop*. |
| EC07 | `flow` node had no error port                                | ADR **D21**: implicit `error` port carrying `{state, failed_asserts, unhandled_errors}`; name `error` reserved on End ports & python outputs (**E012**). Engine §5, schema *Flow node*. |
| EC08 | Empty/degenerate seed → pump deadlocks (no QUIESCENT)        | fix: finalize immediately when post-seed `in_flight == 0`. Engine §3; rationale folded into D14. |
| EC09 | Python worker concurrency unspecified; stuck firing blocks   | **fix 2026-07-13:** serial-per-module remains explicit, but timeout/cancellation now immediately terminates, waits one grace, hard-kills, and reaps before replacement; a synchronized regression proves no late side effect or overlap. | EC10 | Runtime-acquired secrets unmasked; masking algo unspecified  | **OPEN after v0.2 under D35/D39:** preserving raw local truth is intentional, but a login-acquired token still cannot be identified for an optional masked report or future safe export. | EC11 | Check-code numbering: E010 missing, W105 misplaced           | fix: E010 marked retired/reserved; W105 moved into the W-block; CLAUDE.md corrected to E001–E009, E011–E012, W101–W107. Engine §8. |
| EC12 | Templating asymmetry: `trigger` envelope vs `nodes.*` value  | doc: `trigger` = full `{value, meta}` envelope; `nodes.<id>.<port>` = unwrapped value; prefer `trigger`. Engine §6, schema *Templating*; example switched to `trigger.value.body.state`. |
| EC13 | request `error` vs `response` for non-2xx unstated           | doc: non-2xx = valid response on `response`; `error` = transport failures only (connection/DNS/TLS/timeout-after-retries). Engine §5, schema *Request*. |
| EC14 | `napf check` import-vs-AST of `nodes.py` undefined           | decided: `check` AST-parses (no import side effects); the worker imports for real at run time; dynamic signatures invisible to check. Engine §8. |
| EC15 | `loop` input port + `over` evaluation, `start.out` payload   | doc: loop fires on `trigger`, `over` evaluated against that delivery; `start.out` carries the frame's full `inputs` dict. Schema catalog notes, engine §4/§5. |
| EC16 | Counter off-by-one; W101 cycle-vs-SCC check scope            | decided: counter is check-then-decrement — `count: N` = exactly N `continue` passes, message N+1 exhausts. W101 gives the strict guarantee (every simple cycle guarded) in linear time: delete guard nodes, test acyclicity — a remaining cycle is exactly a guard-free cycle. Schema *Guards*, engine §8. |
| EC17 | Env validation split: W105 (check) vs ENV (run)              | doc: W105 warns when a key is in *no* profile at check time; ENV errors when missing from the *active* profile at run time — by design. Engine §2/§8. |
| EC18 | `nodes.*` is last-writer-wins under cycles/concurrency       | doc: stated; prefer `{{ trigger }}` for the firing value. Engine §6. |
| EC19 | Set/Get reintroduce ordering hazards wires avoid             | doc: Set-before-Get holds only when a path exists from the Set to the Get's `trigger`; frame variables are not a synchronization primitive. Schema *Scoping*. |
| EC20 | Abort mid-request leaves dangling `request_started` in JSONL | **handled 2026-07-13:** JSONL readers tolerate one partial/malformed final tail and preserve the valid prefix. M5 freezes each replay request at one sequence/lifecycle snapshot: known-closed prefixes are `incomplete`, external exact `.active` units are `indeterminate`, and only a durable arbitrarily large `run_finished` is `complete`; browser state never invents a finish. Engine §7, `test_incomplete_prefix_and_large_final_record_are_classified`, snapshot-race/store/Playwright regressions. |
| EC21 | Node-id charset constraints unspecified                      | decided: ids match `[A-Za-z_][A-Za-z0-9_]*`; E011 enforces charset + uniqueness. Schema *Node ids*. |
| EC22 | Grandchild processes from python nodes not killed            | **OPEN, explicitly after v0.2:** documentation does not reap them. Needed: spawn workers in an owned POSIX process group / Windows Job Object or equivalent, terminate the whole descendant tree on timeout/abort/shutdown, and add cross-platform child+grandchild survival tests. Until then only the worker process is covered. |
| EC23 | `defaults.request` may only reference `env`/`run`            | doc: `inputs`/`nodes` are frame-scoped and would StrictUndefined-error every inheriting request. Manifest rule 4. |
| EC24 | Nodes without an error port: where do evaluation errors go? (found 2026-07-02) | decided: evaluation/template failures on port-less-error nodes (condition, switch, merge, guards, set/get, delay, log, fixture) are unhandled node errors — recorded in the report, run `failed`. Engine §6. |
| EC25 | `max_seconds` referenced by engine §5a but never documented in the flow schema (found 2026-07-02) | ADR **D24**: universal optional node key; documented in schema *Execution timeouts* with per-node routing (`{error_kind: "timeout"}` on the error port, never a data port). |
| EC26 | Default 300s ceiling applied to container/self-bounded nodes — healthy long loops/subflows killed at 5 min (found 2026-07-02) | ADR **D24**: the default auto-applies to `request`/`python` only; `delay`/`loop`/`flow` exempt from the default (bounded by config/children), explicit `max_seconds` honored; container timeout routing defined (flow → implicit error port; loop → EC24, wrap-in-subflow to branch). |
| EC27 | No wall-clock run deadline — runs bounded only at budget × ceiling; a CI SIGKILL loses the report (found 2026-07-02) | **PARTIAL 2026-07-13:** 128-delivery ready batches yield and check monotonic deadline/abort state; deadline, heartbeat, and abort regressions close M2/TR-13's cooperative-scheduler half without changing budget/quiescence. **OPEN after v0.2:** synchronous Jinja cannot be preempted; full hard-deadline closure remains EC35. |
| EC28 | Worker protocol self-corrupting: user `print()` writes to stdout — the JSON-lines channel — while the spec claimed stderr carries prints (2026-07-02 senior review) | fix: worker `dup()`s the real stdout fd for protocol lines at startup, rebinds `sys.stdout`/`sys.stderr` to capture streams forwarded as log events. | EC29 | Round-trip + diagnostics architecture unstated: re-dumping Pydantic models would delete comments; diagnostics had no file:line (2026-07-02 review) | decided: the loaded CommentedMap is the single write source (models = read-only views); ruamel source marks threaded through validation; every E/W carries file:line + node id + fix hint. `yaml-profile.md` *Write path*, engine §0/§8. |
| EC30 | Flagship example had no create step and polled `{{ run.id }}` — napflow's own run id, not an API job id (2026-07-02 review) | fix: `create` POST node added; poll URL reads `{{ nodes.create.response.body.id }}` (also demos ghost-wires). Schema example. |
| EC31 | Message budget default 10000 trips legitimate data-driven loops (~1k-row fixture × small body ⇒ false red) (2026-07-02 review) | fix: default raised to 100000 — runaway protection, not resource accounting; a tight unguarded cycle still dies in well under a second. Manifest, engine §3, schema *Guards*. |
| EC32 | 10MB valve is per *body* — one big loop can write gigabytes of JSONL per run, ×`history: 20` (2026-07-02 review) | **fix 2026-07-13:** removed both destructive manifest valves, preview/truncation paths, and new `capture_warning` emission. Store-backed streams apply one exhaustive policy to request/response aggregates, message/Log/error values, and frame/End outputs; identical large values deduplicate by hash while runtime data remains complete. The 200 KB Request→Log→End→JSON-report regression resolves byte-identically from one blob. D34, FR-1102/NFR-15/TR-16. |
| EC33 | Windows: asyncio subprocess pipes need the Proactor loop; ASGI/WS stacks sometimes force Selector — python workers would break under `napf ui` only (2026-07-02 review) | doc + test: requirement stated in engine §5a; Windows integration test runs a python-node flow through the server (TR-9). |
| EC34 | First-touch check depended on httpbin.org — offline/proxy/flaky ⇒ broken first impression and flaky CI (2026-07-02 review) | fix: `napf init` adds offline `flows/smoke` (fixture→python→assert) as the first-touch check; httpbin example kept as the HTTP demo. Manifest. |
| EC35 | Trust model undocumented: running a workspace executes its `nodes.py`; Jinja sandbox ≠ security boundary; sync render can stall the loop (2026-07-02 review) | **OPEN accepted limitation, not resolved:** trust documentation is correct, but synchronous template rendering can still freeze the event loop and evade a run deadline. | EC36 | Bundle of small underspecs: `.env` dialect; same-node firing overlap; `parallel` loop `results` ordering; Start `default:` scope; python optional-vs-required params under AST (2026-07-02 review) | doc: dotenv-style dialect, no interpolation (manifest §2); async firings may overlap, sync serialized (engine §4); `results` index-ordered (engine §5, schema); Start defaults see env/run at BIND (engine §6, schema); literal defaults = optional (schema). |
| EC37 | `{{ }}` always renders strings — structured bodies impossible (`body:` would emit Python-repr'd dicts, not JSON) (2026-07-02 review) | ADR **D25**: a config value that is exactly one `{{ expr }}` evaluates natively; mixed content renders to string; field schema type applies post-evaluation. | EC38 | Write endpoints take a flow identity as a path tail — a crafted identity (`..`, absolute path, drive-letter colon) could write outside the workspace (S4/M4) | **fix 2026-07-12:** one `WorkspaceResolver` now owns entry/ref/loop/fixture/history/source/clone paths, validates POSIX lexical identities + run IDs, resolves symlinks, and enforces canonical containment with stable `workspace_boundary` prep/API errors. | EC39 | Flow-detail GET 400'd on check E-codes — a mid-edit flow (e.g. transient E004) locked the canvas, so the editor couldn't fix its own in-progress state; broke the M3 e2e when changed (S4/M4) | fix: check diagnostics never 400 the detail endpoint (flows stay editable, E-codes render as diagnostics); only unloadable files 400; E-codes still gate *runs*. Manifest *Server surface*. |
| EC40 | Rejecting a nodes.py save on syntax errors would hold code hostage over a missing colon — no way to save work-in-progress (S4/M4) | doc: PUT `/api/code/*` always saves (last-write-wins); the syntax error is AST-detected and reported in the response + inline in the editor, never blocks. Manifest *Server surface*. |
| EC41 | An edge wired to a port the node's surface doesn't declare (merge growth, broken python ref ⇒ `null` surface) would orphan visually on the canvas (S4/M3) | fix: the canvas grows an `any`-typed handle for wired-but-undeclared ports — edges never dangle; the checker still owns correctness. | EC42 | Product invariant `from napflow.core import run_flow` is documented but `core.__init__` exports no such API; installed import raises `ImportError`. | **fix 2026-07-13:** functional `run_flow`/`run_flow_async` and immutable Workspace/Flow handles share one prepare/execute/cleanup path. Generated static bindings remain post-v0.2 (D38). FR-1112/TR-11. |
| EC43 | Run lifecycle safety is locally implemented and bypassable: inline ready cycles starve deadlines/abort; external cancellation skips cleanup; worker JSON lines exceed asyncio's default 64KB reader; timeout performs graceful EOF before terminate, allowing late side effects and replacement overlap. | **fix 2026-07-13:** bounded fair pump + monotonic control checks; one shielded cleanup task closes firing/loop tasks, HTTP, workers, streams, and JSONL before cancellation escapes; an ordinary sink-close failure cannot replace the run result or skip CLI/server history abandonment, while `KeyboardInterrupt`/cancellation still propagate; worker abnormal teardown is immediate and reap-before-replace; 16 MiB protocol ceiling routes malformed/oversize failures stably. | EC44 | README's Git install builds from a clean tree without the gitignored UI bundle, so `napf ui` serves a placeholder; release tag can also diverge from package version and frontend unit/e2e are not one authoritative tag gate. | **fix 2026-07-14:** D40 supports release-built PyPI/GitHub wheel or sdist artifacts and rejects the raw VCS/source promise. One reusable PR/tag workflow now gates the three-OS Python/UI suites, Vitest, notices, artifact smoke, and exact tag/`.dev` refusal; release-gate/distribution regressions and PR CI run #42 are green. | EC45 | Secret masking recursively rewrites the entire event and dictionary keys; a declared value such as `passed` or `error` corrupts state/key vocabulary, while irreversible emission-time masking destroys local ground truth. | **fix 2026-07-13:** canonical JSONL/local WebSocket records are raw and the exhaustive event-field registry drives value-only terminal/report redaction, preserving all keys, identifiers, enums, and control fields; `passed`/`error`, raw-history, CLI, JSON/JUnit, and empty-pattern regressions pin the fix. | EC46 | Debounced canvas/code edits are dropped on navigation, editor close, or unload; overlapping saves reuse stale ETags; YAML/nodes.py truncate the live source directly, so interruption/disk-full can corrupt it. | **fix 2026-07-12:** revisioned canvas/code coordinators serialize saves and chain ETags; navigation/editor close flush, unload prompts, conflict/error blocks transitions, and generation guards reject stale responses. | EC47 | History/execution resource handling is unbounded or incorrect: one loop task and retained frame per item; live prefixes/subscriber queues/full REST/UI/CLI buffers grow with the whole run; 64KB tail mislabels large completions; random same-second IDs and start-time retention can delete a new/active log; report companions leak. | **fix 2026-07-13 (M3+M5):** fixed loop workers/frame compaction pass ordering, active-bound, 100k, and abort-tail regressions. | EC48 | Static validation accepts callable/input shapes the worker cannot invoke (`async def`, positional-only arguments); malformed binary request envelopes can become internal engine errors or decode permissively instead of routing a request error. | **fix 2026-07-13:** checker and worker consistently reject async/positional-only callables (positioned E008 + defensive node error); binary envelopes require the exact shape, non-empty content type, and canonical base64, with non-retryable `request_encoding` error-port routing. EC50/M4 later completed TR-21's prepared-request half. |
| EC49 | Visual/API schema transport drifts: raw URL/deep-link identities break on reserved characters, and the editor cannot author universal/template-aware fields such as `max_seconds` and selected typed request/Start values. | **fix 2026-07-13 (M1+M6):** M1 fixed identity transport with per-segment encoding, canonical routes, and reserved-name browser coverage. FR-1111/1114, TR-12/22. |
| EC50 | “Full request detail” is not the prepared wire request: history stores a 512-character body preview before query, cookies, effective headers, and transport preparation. | **fix 2026-07-13:** the niquests adapter snapshots the effective initial and final redirect-aware prepared requests (encoded URL/query, library/session headers, cookies, exact body/no-body and size). Start/finish/failure events carry that request under the full-value/blob policy together with retries, redirects, response detail, and timing. FR-1103/TR-21. |
| EC51 | Loopback binding is treated as the entire browser security boundary; mutation and WebSocket routes do not validate Host/Origin even though the server can rewrite and execute `nodes.py`. | **fix 2026-07-12:** app middleware validates exactly one loopback Host on every request and matching browser Origin on mutations/WebSockets before handler/accept; programmatic loopback clients may omit Origin. | EC52 | Windows treats a dangling `.history.lock` symlink as missing during the first open, so the create fallback can materialize its external target before post-open validation rejects the link. | **fix 2026-07-14:** lock setup rejects a pre-existing non-regular entry via `lstat` before `os.open`, while retaining exclusive creation and post-open descriptor/path identity validation. The cross-platform regression proves a planted symlink is never opened and its target remains absent. Engine §7a. |
| EC53 | The canonical YAML profile promised no anchors/aliases, but the round-trip loader accepted them and the emitter could synthesize them from repeated object identities. | **fix 2026-07-14:** the production composer rejects anchor/alias events with positioned load diagnostics, while quoted `&`/`*` remain data. FR-204/YP. |
| EC54 | JSON/JUnit report generation used a weaker history-header check than REST/WS replay, so explicit `format: null`, a non-`run_started` first record, or an empty/malformed history could bypass the promised format gate. | **fix 2026-07-14:** one core `validate_history_envelope` now serves replay and reports. EN §7a/TR-22. |
